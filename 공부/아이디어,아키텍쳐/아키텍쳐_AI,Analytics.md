# AI 모델이 한번 학습하면 끝인가?
--> 아니오. 계속 관리 해야한다
스마트 팩토리 현장은 환경이 계속 변한다
1. 설비 상태 변화 (새 부품 교체, 마모, 유지보수) -> 센서 패턴 변화
2. 공정 조건 변화 
(생산품 종류 변경, 공정 온도/속도 변경)
3. 데이터 분포 변화
(초기 학습 데이터와 지금 데이터가 달라짐 -> ML 모델 성능 저하)
--> 이런 현상을 Data Drift / Concept Drift 라고 부른다

# 학습 주기
1. offline(배치 학습)
- 하루/주 단위로 새 데이터 모아 재학습
- 장점: 안정적, 계산 부담 적음
- 단점: 최신 이상치 바로 반영 어렵
2. Online/ Incremental
- 실시간 데이터 들어올 때마다 조금씩 학습
- 장점 :최신 데이터 반영
- 단점 : 구현 복잡, 모델 안정성 관리 필요

# 재학습 대상
- Isolation Forest, LOF 
:새 패턴 반영 필요시, 이상치 기준변화 시
- Regression 모델 
:설비 변화 반영시, KPI 예측 정확도 유지 필요시
- Classification 모델 
:상태 변화 반영시, 위험도 분류 정확 유지 필요시

# 현업 적용 방법
1. 모델 모니터링
- KPI 오차, 이상치 탐지율 지속 체크
- 성능 낮으면 재학습 트리거
2. 주기적 재학습
- 주 1회, 월 1회 등 운영 정책에 따라 수행
